## Spacing Scale

Declared values (must be multiples of 4) — already the de facto scale in use across `app/morpho/page.tsx`, `app/dashboard/page.tsx`, `app/page.tsx` (`p-4`, `p-6`, `px-4 py-2`, `px-6 py-3`, `gap-4`, `gap-6`, `mb-2/3/4/8`, `space-y-2/4/6`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge inline padding |
| sm | 8px | Compact spacing inside a stat row (`space-y-2`) |
| md | 16px | Default card padding (`p-4`), inline gaps (`gap-4`) |
| lg | 24px | Card padding for content-heavy cards (`p-6`), section `gap-6` |
| xl | 32px | Between major page sections |
| 2xl | 48px | Major section breaks (rare, e.g. hero-to-grid spacing on `/`) |
| 3xl | 64px | Not currently used anywhere in the app; available if a full-page empty state needs top padding |

Exceptions: none for this phase. No icon-only touch targets are planned (all buttons/links carry text labels, matching every existing page).

## Typography

Declared from the sizes/weights already observed in production pages (`text-xs/sm/lg/xl/2xl/3xl`, `font-medium/semibold/bold`):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (`text-xs`) | 600 (`font-semibold`) | 1.2 |
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 |
| Display | 24px (`text-2xl`) | 700 (`font-bold`) | 1.2 |

Weight budget: **2 weights for anything new in this phase — 400 (body) and 600 (labels, card/section headings)**. 700 (`font-bold`) is reserved exclusively for the fixed `bg-onyx` page-header `<h1>` — this is a pre-existing, app-wide layout element (identical markup already in `/morpho`, `/dashboard`, `/`), not a new weight introduced by this phase. Do not use `font-bold` anywhere else on `/goals`.

Card titles (`<h3>` in existing cards, e.g. `app/dashboard/page.tsx:112`) use `text-lg` (18px) at `font-semibold` — treat this as an acceptable variant of the Heading role for card-level (not section-level) titles; do not add a 5th distinct size.

## Color

60/30/10 split, using only existing tokens from `app/globals.css` — **do not invent new hex values or CSS variables**:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `#faf9f6` (`bg-background`, or default body bg) | Page background behind all cards |
| Secondary (30%) | `--onyx` `#0b0b0d` (header bar) + white (`bg-white`) card surfaces bordered by `--gold-soft` `#e9dcae` (`border-gold-soft`) | Page header, all content cards, table zebra dividers (`divide-gold-soft`) |
| Accent (10%) | `--gold` `#c9a227` (`text-gold`, `border-gold`, `bg-gold-soft/20`) | Reserved for: (1) the smoothed trend line and target-value reference line on the weight/body-fat chart, (2) `border-l-4 border-gold` callout accents (mirroring `/morpho`'s segment list), (3) active-state text/hover on nav links (`hover:text-gold`), (4) small uppercase eyebrow labels (`text-gold-dark`, mirroring `/morpho`'s "Semaine du..." label). Never use gold as a full button fill — that role belongs to crimson. |
| Destructive | `--crimson` `#a8171f` (`bg-crimson`, `text-crimson`) — **no separate destructive red exists in the palette; this app has never had a delete/destructive action.** If the planner adds a "supprimer l'objectif" affordance, reuse `--crimson` for the button, exactly as it's already used for `bg-crimson` primary CTAs (`/morpho`'s "Lancer l'analyse hebdo", `/`'s "Synchroniser") — there is no visual differentiation available between "primary action" and "destructive action" in this palette today, so destructive actions MUST get a text-based confirmation step (see Copywriting) to compensate. |

Accent reserved for: trend-chart smoothed line + target reference line, `border-l-4` callouts, uppercase eyebrow labels, active/hover link states. Never apply gold to body text, card backgrounds, or more than one emphasis element per card (matches existing restraint — e.g. `/morpho` uses at most one gold accent per card).

`--crimson` is simultaneously "primary action" (buttons) and "destructive" in this app's existing palette — this is a pre-existing constraint, not a new decision for this phase.

## Phase 1 Visual Notes (chart & data-view contract)

These are prescriptive requirements specific to GOAL-02/03/04 and directly enforce PITFALLS.md pitfalls 5, 6, 7 and CONTEXT.md D-03/D-04:

**Weight/body-fat trend chart (GOAL-02):**
- Library: `recharts@^3.9.1` `<ResponsiveContainer>` + `<LineChart>`.
- Primary line: smoothed trend (7-day rolling average — Claude's Discretion per CONTEXT.md, document the chosen method in code comments), stroke `--crimson`, `strokeWidth={2.5}`, drawn on top.
- Secondary layer: raw daily measurement points, rendered small/light — either a `<Scatter>`/`<Line dot>` layer with low-opacity `--ink-soft` (e.g. `stroke-ink-soft/40`, no connecting line, small dot radius) or a thin faint line beneath the smoothed one. Must read as clearly secondary, never equal visual weight to the trend line (per D-04 and Pitfall 5).
- Target value: a horizontal `<ReferenceLine>` at the goal's target weight/body-fat value, dashed, `stroke="var(--gold)"`, with a small label.
- Target date: text only, next to the goal summary (e.g. "Objectif : 75 kg — pour le 15 déc. 2026"), never drawn as a line/marker on the chart's x-axis (per CONTEXT.md discretion note and Pitfall 7 — no linear date-based target line).
- Legend/labeling: explicitly label which line is "tendance" vs. raw points, so the two are never visually confused (Pitfall 5 warning sign).
- Two independent chart instances (or a toggle) for weight and body-fat % — do not force both onto one dual-axis chart if it hurts readability; a simple stacked pair of charts is preferred over a cramped dual-axis chart, consistent with this app's plain, readable card style.

**Strength/volume progression (GOAL-03):**
- Scope: top 5-8 exercises by session frequency (D-03), reusing the aggregation logic already in `buildHevyContext()` (`lib/morpho.ts`) — do not build a full exercise picker/dropdown for v1.
- Layout: one small chart per exercise in a card grid (mirrors the existing `grid grid-cols-1 md:grid-cols-2/3 gap-4/6` pattern already used for photos/segments in `/morpho`), not one giant combined chart.
- Chart type: line or bar showing best working set weight (or volume) per session over time, `--crimson` as the primary series color, `--gold-soft` gridlines to match the app's border tone.
- Card header per exercise: `text-lg font-semibold text-ink` (matches existing card-title convention).

**Progress-photo timeline (GOAL-04):**
- Layout: chronological, scrollable list/grid — reuse the existing 3-column photo grid pattern from `/morpho` (`grid grid-cols-1 md:grid-cols-3 gap-4`, `<img className="w-full h-64 object-cover rounded" />`) but repeated per week/entry down the page, most recent first.
- Each entry gets a lightweight date header (`text-xs uppercase tracking-wide text-gold-dark font-semibold`, matching `/morpho`'s "Semaine du..." eyebrow label) above its photo row.
- No compare-slider in this phase (GOAL-07 is v2/deferred) — simple chronological gallery only.

**Goal-model framing (Pitfall 6 — avoid weight-loss/fat-loss conflation):**
- The `/goals` page must show weight trend, body-fat trend, AND the strength/volume chart together on one page (not weight alone as "the" progress indicator) — this is already satisfied structurally by GOAL-02+GOAL-03+GOAL-04 all living on the same `/goals` page (D-01), but the layout must not visually bury the strength/photo evidence below an oversized weight chart. Recommended order top-to-bottom: goal summary (target values + target date as text) → weight/body-fat trend charts → strength/volume charts → photo timeline.

**Page chrome (matches every existing page):**
- Header: `bg-onyx border-b-2 border-gold`, inner `max-w-5xl mx-auto px-4 py-4 flex justify-between items-center`, `<h1 className="text-2xl font-bold text-white">` + "Accueil" link back to `/` (`text-white/70 hover:text-gold underline`) — copy exact structure from `app/morpho/page.tsx:61-66`.
- Main content: `max-w-5xl mx-auto px-4 py-8`, cards as `bg-white border border-gold-soft rounded-lg p-6` (or `border-2 border-gold` for the top-level summary card, matching `/morpho`'s summary-header treatment).
- Home page (`app/page.tsx`) card: insert a new `<Link href="/goals">` card into the existing grid using the identical card markup (`bg-white p-6 rounded-lg border border-gold-soft hover:border-crimson transition cursor-pointer`, `text-3xl mb-3` emoji, `text-lg font-semibold text-ink mb-2` title, `text-sm text-ink-soft` subtitle) — emoji **🎯**, title **"Objectifs"**, subtitle **"Suivi visuel de tes objectifs poids/masse grasse"**.
